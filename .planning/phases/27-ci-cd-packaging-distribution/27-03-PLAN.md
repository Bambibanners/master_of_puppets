<task type="checkpoint:human-verify" gate="blocking">
  <name>Task 2: Configure PyPI Trusted Publisher and GitHub Environments</name>
  <what-built>
Plan 27-01 created the GitHub Actions release workflow at `.github/workflows/release.yml`. This workflow publishes the `axiom-sdk` Python package to PyPI via OIDC Trusted Publisher on every `v*` tag push. Before the first tag can succeed, you must complete three one-time manual steps on external services.
  </what-built>
  <how-to-verify>
Complete the following steps in order. These cannot be automated — they require clicking through web UIs on external services.

**Step 1: Verify the GHCR image path**

The release workflow hardcodes the image as `ghcr.io/axiom-laboratories/axiom`. The `docker-release` job authenticates with `GITHUB_TOKEN` scoped to the repository owner.

- If your repository lives at `github.com/axiom-laboratories/master_of_puppets` (or similar under `axiom-laboratories`): you're good, no change needed.
- If it lives under a personal account (e.g., `github.com/thomas/master_of_puppets`): open `.github/workflows/release.yml` and change `ghcr.io/axiom-laboratories/axiom` to `ghcr.io/<your-github-username>/axiom`. The `packages: write` permission uses `GITHUB_TOKEN` which is always scoped to the repository owner automatically.

**Step 2: Create GitHub Environments**

In your GitHub repository: **Settings → Environments → New environment**

Create two environments:
1. Name: `testpypi` — no required reviewers needed
2. Name: `pypi` — optionally add yourself as a required reviewer for extra safety

**Step 3: Configure TestPyPI Trusted Publisher**

1. Go to https://test.pypi.org and log in (create an account if needed)
2. Create the `axiom-sdk` project if it doesn't exist yet (upload a dummy distribution, or just set up the Trusted Publisher record on an existing project)
3. Navigate to the project → **Publishing** → **Add a new publisher**
4. Fill in:
   - PyPI project name: `axiom-sdk`
   - Owner: `axiom-laboratories` (or your actual GitHub org/user owning the repo)
   - Repository name: `master_of_puppets` (or your actual repo name — check `git remote get-url origin`)
   - Workflow name: `release.yml`
   - Environment name: `testpypi`
5. Save.

**Step 4: Configure PyPI Trusted Publisher**

1. Go to https://pypi.org and log in
2. Create the `axiom-sdk` project (same process as TestPyPI if not done)
3. Navigate to the project → **Publishing** → **Add a new publisher**
4. Fill in exactly as above but with **Environment name: `pypi`**
5. Save.

**Verify the setup is ready:**

- [ ] `.github/workflows/release.yml` exists and GHCR image path matches your repository owner
- [ ] GitHub environment `testpypi` created in repository Settings → Environments
- [ ] GitHub environment `pypi` created in repository Settings → Environments
- [ ] TestPyPI Trusted Publisher record added (repo + release.yml + testpypi env)
- [ ] PyPI Trusted Publisher record added (repo + release.yml + pypi env)

Once these are done, the release pipeline is ready. To trigger your first release:
```bash
# 1. Edit pyproject.toml version if needed (currently 1.0.0-alpha → PEP 440: 1.0.0a1)
# 2. Commit and push to main
# 3. Tag and push the tag
git tag v1.0.0-alpha
git push origin v1.0.0-alpha
```

The release workflow will run, build the multi-arch Docker image, push to GHCR, build the Python package, publish to TestPyPI, then publish to real PyPI.
