## Milestone Summary

**Key Decisions:**
- Soft-delete over hard-delete for CapabilityMatrix — preserves history, reversible, admin can view inactive entries
- JSON string storage for runtime_dependencies (TEXT column) — consistent with existing target_tags pattern; avoids JSON column type SQLite doesn't natively support
- STRICT mode returns HTTP 403 blocking the build; WARNING mode sets is_compliant=False on template and continues
- smelter_enforcement_mode stored in Config table (key/value) — no new table needed
- Mirror fail-fast raises 403 unconditionally — enforcement_mode only gates unapproved-ingredients check (separate concern)
- Soft-purge (is_active=False) used for ingredient delete — preserves mirror files and history
- Upload routing by file extension (.deb → apt, else → pypi) — replaces os_family-based routing
- Phase 16 (Security & Governance) descoped — deferred to a future milestone

**Issues Resolved:**
- Mirror test suite required pyproject.toml pythonpath=['puppeteer'] for agent_service imports to resolve
- Sequential side_effect list in mock DB used instead of SQL-repr string matching (bound parameters make literal search unreliable)
- Phase 13 required 3 gap-closure plans after verification revealed frontend and test gaps
- Phase 12 required 1 gap-closure plan for SMLT-04 mirror-status enforcement

**Technical Debt Incurred:**
- Phase 16 (SLSA provenance, Ed25519-signed build provenance, --secret credentials) deferred
- MIN-6 (SQLite NodeStats pruning compat), MIN-7 (foundry build dir cleanup), MIN-8 (per-request DB query in require_permission), WARN-8 (non-deterministic node ID scan) remain deferred
