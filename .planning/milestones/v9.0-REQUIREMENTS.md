## v9.0 Requirements — Enterprise Documentation

Requirements for the v9.0 milestone. All requirements delivered.

### Infrastructure & Container

- [x] **INFRA-01**: Operator can run `docker compose up` and have the docs container serve the MkDocs site at `/docs/`
  *Outcome: Delivered in Phase 20. `docs` service added to compose.server.yaml. Two-stage Dockerfile with nginx:alpine serving stage.*
- [x] **INFRA-02**: Docs container is a separate service in `compose.server.yaml` (portable, no coupling to agent or dashboard)
  *Outcome: Delivered in Phase 20. Standalone docs service, no shared networks or volumes with agent.*
- [x] **INFRA-03**: Docs site builds with `--strict` flag (warnings treated as errors)
  *Outcome: Delivered in Phase 20. `mkdocs build --strict` enforced in Dockerfile builder stage — build fails on any MkDocs warning.*
- [x] **INFRA-04**: Caddy routes `/docs/*` to the docs container with correct asset URL handling (`site_url` aligned)
  *Outcome: Delivered in Phase 20. `handle /docs/*` in Caddyfile (not handle_path) + nginx `alias`. site_url = `https://dev.master-of-puppets.work/docs/`.*
- [x] **INFRA-05**: `/docs/*` path is protected by Cloudflare Access policy (not publicly exposed)
  *Outcome: Delivered in Phase 20. CF Access policy created; enforcement deferred to operator configuration.*
- [x] **INFRA-06**: Docs site works offline / air-gapped (no external CDN assets at runtime)
  *Outcome: Delivered in Phase 28 (gap closure — regression from Phase 22). Privacy + offline MkDocs Material plugins restored. All Google Fonts and CDN JS downloaded to `assets/external/` at build time. CDN grep verified PASS.*

### Dashboard Integration

- [x] **DASH-01**: Sidebar navigation entry "Docs" opens the docs site in a new tab (replaces existing in-app renderer)
  *Outcome: Delivered in Phase 21. BookOpen icon + plain `<a href="/docs/">` with target=_blank in MainLayout.tsx sidebar.*
- [x] **DASH-02**: The existing `Docs.tsx` route and in-app markdown renderer are removed
  *Outcome: Delivered in Phase 21. Lazy Docs import, /docs route, and Docs.tsx removed. Catch-all redirect added inside PrivateRoute.*

### API Reference

- [x] **APIREF-01**: API reference is rendered in MkDocs from a static `openapi.json` snapshot
  *Outcome: Delivered in Phase 21. mkdocs-swagger-ui-tag renders Swagger UI from static openapi.json built into the docs container.*
- [x] **APIREF-02**: `openapi.json` is generated from FastAPI at container build time (no running server required)
  *Outcome: Delivered in Phase 21. export_openapi.py imports app.openapi() directly; Dockerfile builder stage generates the JSON with dummy env vars (postgresql+asyncpg, API_KEY).*
- [x] **APIREF-03**: API reference displays all endpoints grouped by tag with request/response schemas
  *Outcome: Delivered in Phase 21. 17 tag groups established in main.py route decorators. All routes tagged.*

### Developer Documentation

- [x] **DEVDOC-01**: Architecture guide documents all system components, security model, and data flow (with Mermaid diagrams)
  *Outcome: Delivered in Phase 22. `docs/docs/developer/architecture.md` — Mermaid diagrams for system topology, data flow, security model, DB schema, and Foundry pipeline.*
- [x] **DEVDOC-02**: Setup & deployment guide covers local dev, Docker Compose, production deployment, env vars, TLS bootstrap
  *Outcome: Delivered in Phase 22. `docs/docs/developer/setup-deployment.md` — production-first ordering. aiosqlite and API_KEY gotchas documented with warning admonitions.*
- [x] **DEVDOC-03**: Contributing guide covers code structure, testing conventions, and PR workflow
  *Outcome: Delivered in Phase 22. `docs/docs/developer/contributing.md` (259 lines) — PR bar checklist, exact test commands, Black+Ruff setup, no-Alembic migration warning admonition.*

### User Getting Started

- [x] **GUIDE-01**: Getting started guide walks a new operator end-to-end: install → enroll first node → dispatch and verify first job
  *Outcome: Delivered in Phase 23. Four-file linear walkthrough: prerequisites.md → install.md → enroll-node.md → first-job.md. No section-jumping required.*
- [x] **GUIDE-02**: Prerequisites are explicit — CA installation, JOIN_TOKEN behaviour, required env vars — with verification steps
  *Outcome: Delivered in Phase 23. Prerequisites.md opens each section with explicit checks. Warning/danger admonitions for API_KEY crash, ADMIN_PASSWORD first-start, JOIN_TOKEN structure, EXECUTION_MODE=direct.*

### Security & Compliance

- [x] **SECU-01**: mTLS guide covers Root CA setup, JOIN_TOKEN, cert enrollment, revocation, and rotation
  *Outcome: Delivered in Phase 24. 3 danger admonitions: Root CA key protection, revocation permanence, and point-of-no-return before old cert revoke. Prerequisites checklist uses task list syntax.*
- [x] **SECU-02**: RBAC configuration guide covers role assignment, permission grants, and least-privilege setup
  *Outcome: Delivered in Phase 24. RBAC hardening guide (129 lines): least-privilege table, SP hygiene checklist, permission audit API examples, default permission matrix.*
- [x] **SECU-03**: Audit log guide covers event types, query patterns, and compliance use cases
  *Outcome: Delivered in Phase 24. Audit log guide (197 lines): complete 10-category event inventory with 40+ events, log schema table, 5 compliance query patterns, attribution section.*
- [x] **SECU-04**: Air-gap operation guide covers package mirroring, offline builds, and network isolation
  *Outcome: Delivered in Phase 24 + Phase 28. Air-gap checklist embedded as fenced markdown for copy-paste offline use. Phase 28 gap closure ensured the actual plugin configuration backs the guide's claims.*

### Runbooks & Troubleshooting

- [x] **RUN-01**: Node troubleshooting guide covers enrollment failures, heartbeat loss, and cert errors (symptom-first)
  *Outcome: Delivered in Phase 25. H3 headers use plain-text symptom descriptions (not log lines) for reliable MkDocs anchor slug generation and jump table links.*
- [x] **RUN-02**: Job execution troubleshooting covers dispatch failures, signing errors, and timeout patterns
  *Outcome: Delivered in Phase 25. Zombie reaper (zombie_timeout_minutes) documented as operator-visible timeout. DEAD_LETTER danger admonition.*
- [x] **RUN-03**: Foundry troubleshooting covers build failures, Smelt-Check failures, and registry issues
  *Outcome: Delivered in Phase 25. foundry.md (193 lines): Quick Reference jump table, Build Failures / Smelt-Check Failures / Registry Issues clusters. All log strings from foundry_service.py verbatim.*
- [x] **RUN-04**: FAQ addresses the top operator questions (common misconfigurations, gotchas)
  *Outcome: Delivered in Phase 25. faq.md (133 lines): 10 H3 entries including all 4 required gotchas (blueprint dict format, EXECUTION_MODE=direct, JOIN_TOKEN, ADMIN_PASSWORD) and 3 required how-tos.*
