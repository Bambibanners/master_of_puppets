# Milestone v9.0: Enterprise Documentation

**Status:** ✅ SHIPPED 2026-03-17
**Phases:** 20–28 (9 phases)
**Total Plans:** 27

## Overview

Brought all technical and user documentation to enterprise standard, hosted as a containerised MkDocs Material wiki within the stack and linked from the dashboard. The docs site runs at `/docs/`, is fully air-gapped (no runtime CDN calls), and covers every operator and developer workflow from getting started through security hardening and troubleshooting.

In parallel, the project was rebranded to "Axiom": CLI renamed from `mop-push` to `axiom-push`, README rewritten, CONTRIBUTING + CHANGELOG + GitHub community health files established, and CI/CD pipelines created for GHCR/PyPI release automation.

## Milestone Summary

**Key Decisions:**

- MkDocs Material chosen — git-backed markdown, no DB, static build, portable, all Insider features free (9.7.0+)
- Two-stage Dockerfile (python:3.12-slim builder + nginx:alpine serve) — mkdocs serve is not production-safe
- Caddy must use `handle /docs/*` (NOT handle_path) + nginx `alias` — prefix stripping silently breaks all CSS/JS assets
- site_url must be `https://dev.master-of-puppets.work/docs/` — bakes subpath into all asset references
- openapi.json generated via app.openapi() import at build time (no running server) — dummy env vars required in builder stage
- Nav is task/audience-oriented (Getting Started / Feature Guides / Security / Developer / API Reference) — must not be restructured after content is written
- CDN verification uses https:// prefix match — privacy plugin reuses domain as local asset directory path
- CLI renamed from `mop-push` to `axiom-push` throughout all 21 docs files, pyproject.toml, and installer scripts

**Issues Resolved:**

- INFRA-06 gap closure — Phase 22 regression commit (ab25961) removed privacy + offline plugins; restored in Phase 28
- CF Access policy scope decision — all /docs/* gated (phase 20)
- SQLAlchemy async engine import side effects at openapi.json build time — resolved with dummy env vars in Dockerfile builder stage

**Issues Deferred:**

- PyPI Trusted Publisher setup — GitHub org `axiom-laboratories` and PyPI project `axiom-sdk` do not yet exist; CI workflows are ready and waiting
- CF Access policy enforcement deferred to future session — local routing confirmed working

**Technical Debt Incurred:**

- Black/Ruff not run on existing code — pyproject.toml added as config only; code style enforcement deferred to separate PR
- mkdocs build --strict cannot pass via local CLI (missing openapi.json) — only passes via Docker builder stage; pre-existing limitation from Phase 21
