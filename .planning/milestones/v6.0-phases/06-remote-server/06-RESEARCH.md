# Phase 06-01 Research: Remote Server & SSL

## Current State Analysis

### 1. Caddy & SANs
- The `Caddyfile` currently uses: `tls /etc/certs/caddy.crt /etc/certs/caddy.key`.
- These certs are generated by the `cert-manager` service.
- We need to verify if `cert-manager` (which uses `agent_service/pki.py`) includes the `SERVER_HOSTNAME` environment variable in the Certificate's SANs.

### 2. Networking
- Caddy listens on 80 and 443.
- All internal communication (`agent` <-> `model`) uses the Docker network.
- External nodes must reach the IP of the host machine.

### 3. Deployment Script
- Currently, there is no generic "Setup Server" script.
- The user has to manually clone, set `.env`, and run `docker compose`.

## Research Tasks
1.  Examine `puppeteer/agent_service/pki.py` to see how root and leaf certs are generated and if they support SANs for remote IPs.
2.  Verify `puppeteer/cert-manager/entrypoint.sh` (if it exists) or the Dockerfile to see how it initializes the certs.
3.  Check `puppeteer/Containerfile.server` to ensure all needed tools for remote deployment are present.
