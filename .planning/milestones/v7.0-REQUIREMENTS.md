# Requirements Archive: v7.0 — Advanced Foundry & Smelter

**Archived:** 2026-03-16
**Status:** All requirements validated and shipped

Note: The original REQUIREMENTS.md for this milestone was deleted when v8.0 was archived (v8.0 was developed and shipped concurrently). This archive is reconstructed from ROADMAP.md phase details and phase SUMMARY.md files.

---

## Requirements

### Compatibility Engine (COMP)

- [x] **COMP-01** — OS Family badges visible in Tools table and on RUNTIME blueprint cards
- [x] **COMP-02** — Add Tool dialog with all fields; runtime deps column rendered
- [x] **COMP-03** — Blueprint creation validates OS compatibility; mismatch rejected with offending_tools list
- [x] **COMP-04** — OS Family dropdown present in blueprint creation; tool list re-filters on DEBIAN/ALPINE switch

### Smelter Registry (SMLT)

- [x] **SMLT-01** — Admin can add ingredients (name, version, package_type, description) to the catalog
- [x] **SMLT-02** — pip-audit CVE scan runs on approved ingredients; CVE results stored and displayed
- [x] **SMLT-03** — Blueprint creation validates all packages against the smelter catalog
- [x] **SMLT-04** — Mirror status enforcement: STRICT mode blocks build if ingredient not mirrored; WARNING mode sets is_compliant=False
- [x] **SMLT-05** — Template compliance badging: templates show compliance status based on their blueprint ingredients

### Package Repository Mirroring (PKG / REPO)

- [x] **PKG-01** — Auto-sync: mirror a package to local storage automatically when it is added to the Smelter Catalog
- [x] **PKG-02** — Status tracking: approved_ingredients.mirror_status tracks PENDING / MIRRORED / FAILED per ingredient
- [x] **PKG-03** — Air-gapped manual upload: admins can upload .whl / .deb files directly to the mirror
- [x] **REPO-01** — Mirror sidecars: pypiserver + Caddy APT sidecar run as Docker Compose services sharing a mirror-data volume; health endpoint returns sidecar uptime and disk usage
- [x] **REPO-02** — Fail-fast enforcement: Foundry build raises HTTP 403 if any blueprint package is approved but not yet mirrored
- [x] **REPO-03** — pip redirect: Foundry build injects pip.conf so pip uses the local pypiserver
- [x] **REPO-04** — APT redirect: Foundry build injects sources.list (DEBIAN builds only) so apt uses the local Caddy-hosted APT repo

### Foundry Wizard UI (WIZ)

- [x] **WIZ-01** — 5-step guided wizard replaces raw JSON blueprint editing (Identity → Base Image → Ingredients → Tools → Review)
- [x] **WIZ-02** — Real-time OS filtering: tool list and base image list filter by selected OS family throughout wizard
- [x] **WIZ-03** — Smelter Registry integration: ingredients step shows catalog with mirror status; only approved ingredients selectable

### Smelt-Check (SMCK)

- [x] **SMCK-01** — Post-build ephemeral validation container spawned with resource limits; exit code captured
- [x] **SMCK-02** — Build marked as failed if smelt-check container exits non-zero

### Bill of Materials (BOM)

- [x] **BOM-01** — JSON BOM generated for every successful build (image ID, packages, versions)
- [x] **BOM-02** — BOM stored in image_boms table linked to puppet_template
- [x] **BOM-03** — Package index (package_index table) enables fleet-wide BOM search by package name

### Lifecycle (LCY)

- [x] **LCY-01** — Image status field: ACTIVE / DEPRECATED / REVOKED
- [x] **LCY-02** — Enrollment blocked for nodes attempting to use REVOKED images
- [x] **LCY-03** — Work pull blocked (concurrency set to 0) for nodes running REVOKED images

### Deferred (Not shipped in v7.0)

- [ ] **SEC-01** — SLSA provenance record per build (Phase 16, deferred)
- [ ] **SEC-02** — Ed25519-signed build provenance (Phase 16, deferred)
- [ ] **SEC-03** — Resource limits enforced at build time (Phase 16, deferred)
- [ ] **SEC-04** — --secret credentials: secrets never appear in image history (Phase 16, deferred)
