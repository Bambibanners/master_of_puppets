From puppeteer/cert-manager/Caddyfile (:443 block, abbreviated):
```caddyfile
:443 {
    tls /etc/certs/caddy.crt /etc/certs/caddy.key

    handle /api/* {
        uri strip_prefix /api
        reverse_proxy https://agent:8001 { ... }
    }

    handle /auth/* {
        reverse_proxy https://agent:8001 { ... }
    }

    handle /ws {
        reverse_proxy https://agent:8001 { ... }
    }

    # ← INSERT handle /docs/* HERE (before the fallback below)

    # Dashboard fallback — MUST remain LAST
    handle {
        reverse_proxy dashboard:80
    }
}
```

The :80 block has the same structure. Insert handle /docs/* in BOTH blocks.

CRITICAL routing rule (from STATE.md and RESEARCH.md):
- Use `handle /docs/*` — NOT `handle_path /docs/*`
- `handle_path` strips the /docs/ prefix before proxying → nginx receives /assets/... → no match → 404
- `handle` preserves the full URI → nginx receives /docs/assets/... → alias maps it correctly

```caddyfile

```

The :80 block already has several specific handles (system/root-ca*, /api/*, /auth/*, /ws). Insert the /docs/* block in the same position — before the final `handle` fallback.
