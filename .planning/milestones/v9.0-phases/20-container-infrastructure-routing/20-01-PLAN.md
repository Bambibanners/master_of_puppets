<task type="auto">
  <name>Task 2: Create Dockerfile and nginx.conf; add docs service to compose.server.yaml</name>
  <files>docs/Dockerfile, docs/nginx.conf, puppeteer/compose.server.yaml</files>
  <action>
**docs/Dockerfile** — two-stage build (python builder + nginx serve):
```dockerfile
FROM python:3.12-slim AS builder
WORKDIR /docs
COPY requirements.txt .
RUN pip install --no-cache-dir -r requirements.txt
COPY . .
RUN mkdocs build --strict

FROM nginx:alpine
COPY --from=builder /docs/site /usr/share/nginx/html
COPY nginx.conf /etc/nginx/conf.d/default.conf
EXPOSE 80
CMD ["nginx", "-g", "daemon off;"]
```
Builder stage runs `mkdocs build --strict` — any MkDocs warning fails the Docker build. The serve stage copies only the static `/site` output; no Python runtime in production.

**docs/nginx.conf** — subpath alias config. CRITICAL: trailing slash on BOTH the location path AND the alias path. Without both trailing slashes nginx constructs a wrong filesystem path:
```nginx
server {
    listen 80;
    location /docs/ {
        alias /usr/share/nginx/html/;
        index index.html;
        try_files $uri $uri/ $uri.html =404;
    }
}
```
Do NOT use `root` here — `root` appends the full URI (`/docs/assets/main.css` → `/usr/share/nginx/html/docs/assets/main.css`) which does not exist. `alias` replaces the `/docs/` prefix, mapping correctly to `/usr/share/nginx/html/assets/main.css`.

**puppeteer/compose.server.yaml** — add `docs` service following the existing `dashboard` service pattern. Insert after the `dashboard` service block, before the `registry` service:
```yaml
  docs:
    image: localhost/master-of-puppets-docs:v1
    build:
      context: ../docs
      dockerfile: Dockerfile
    restart: always
```
No exposed ports (Caddy proxies via service name). No volumes (static files baked into image). No depends_on (docs has no runtime dependency on agent or DB).
